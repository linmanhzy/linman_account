# 免费域名与 DDNS 进阶指南（dynv6.net · 零基础版）

> **适用对象**：完全不懂网络的小白用户。
> **目标**：不花一分钱，把一个好记的名字（如 `linman.dynv6.net`）指向你的服务器公网 IP，从而替代难看的 `http://IP地址:8080`。
> **配套项目**：本文以「林蛮记账」为例（后端 Spring Boot + 前端 React/nginx + 安卓 App），但前 4 步是通用知识，任何服务都能用。

---

## 〇、先搞懂几个基础概念（大白话）

在动手前，先把这几个词弄明白，后面就不晕了。

| 词 | 大白话解释 |
|----|-----------|
| **公网 IP** | 你在互联网上的"门牌号"。别人（任何有网的地方）凭这个号码能找到你的机器。形如 `123.45.67.89`。 |
| **内网 IP** | 你家里/公司局域网内部的"房间号"，只在局域网内有效，外面的人凭这个找不到你。形如 `192.168.x.x`、`10.x.x.x`、`172.16.x.x`。 |
| **域名** | 给 IP 起的好记名字，比如 `baidu.com`。人记名字比记数字容易。 |
| **DNS** | 一个"翻译器"：把域名翻译成 IP。你输 `baidu.com`，DNS 告诉你它对应哪个 IP，浏览器才连得过去。 |
| **DDNS（动态域名解析）** | 普通 DNS 指向的 IP 是固定的；但如果你的公网 IP **会经常变**（比如家庭宽带），就需要 DDNS：IP 一变，自动把你的域名重新指向新 IP，别人永远用同一个名字就能找到你。 |
| **A 记录 / IPv4** | 域名 → IPv4 地址 的这条解析记录，就叫 A 记录。本文主要配它。 |

**本项目里"名字"会出现在哪？**
- 安卓 App 里用户**看不到**地址（`VITE_API_BASE` 是写死在安装包里的后台配置）。
- **Web 端用户**在浏览器地址栏会看到，所以"难看"主要影响 Web 端。用 dynv6 免费域名后，Web 端显示 `http://linman.dynv6.net:8080`，比纯 IP 顺眼。

## 一、步骤 1：检测并获取你当前的公网 IP

先搞清楚"我现在上网用的公网 IP 到底是什么"。两种最简单的方法：

### 方法 1：浏览器查（最省事）
1. 打开任意浏览器，搜索框输入 `IP` 或访问 `https://ip.sb`、`https://ip.cn`。
2. 页面会直接显示你当前的**公网 IP**，形如 `123.45.67.89`。
3. 把这个数字记下来（这就是你"上网出口"的公网 IP）。

### 方法 2：命令行查（更精确）
1. 电脑上按 `Win + R`，输入 `cmd` 回车，打开黑窗口。
2. 输入下面任意一行，回车：
   ```
   curl ip.sb
   ```
   或
   ```
   curl ip.cn
   ```
3. 返回的一串数字就是你的公网 IP。

> **小提示**：家庭宽带的这个公网 IP 可能**每天/每次拨号都变**（动态）；云服务器的公网 IP 一般是**固定不变**的。这正是后面要不要装 DDNS 客户端的分水岭。

---

## 二、步骤 2：公网 IP 与服务器 IP 的区别及关联方式

很多人卡在这里，务必看明白。

### 2.1 两种典型场景

**场景 A：你有一台云服务器 / 有固定公网 IP 的机器**
- 这台机器自己就有一个**公网 IP**（比如云厂商给你的 `123.45.67.89`）。
- 域名直接指向这个 IP 即可，**不需要端口映射、不需要 DDNS 客户端**（IP 不变）。
- ✅ 这是最省事的场景。

**场景 B：你用家里电脑/宽带当服务器**
- 你家**路由器（光猫）**有一个公网 IP（就是步骤 1 查到的那个）。
- 但你家**电脑**是内网 IP（如 `192.168.1.100`），外面的人凭 `192.168.1.100` 找不到它。
- 必须做**端口映射**：在路由器上设"外面访问我的公网 IP:8080，就转发到内网电脑 192.168.1.100:8080"。
- 而且家庭公网 IP 常变 → 需要 **DDNS 客户端**自动更新域名指向。

### 2.2 如何判断自己是不是"有公网 IP"

1. 用步骤 1 的方法查到公网 IP（假设 `123.45.67.89`）。
2. 登录你的路由器后台（通常浏览器开 `192.168.1.1` 或 `192.168.0.1`，账号密码在路由器背面）。
3. 找到「网络状态 / WAN 口状态」，看 **WAN 口 IP**。
   - 如果 WAN 口 IP = 步骤 1 查到的公网 IP → **你确实有公网 IP**（场景 A 或场景 B 都可能是，看连的是哪台机器）。
   - 如果 WAN 口 IP 是 `100.64.x.x` 或 `192.168.x.x` 之类 → 你被运营商做了"内网穿透/NAT"，**没有真正的公网 IP**，本文的直连方案不适用（需走内网穿透 frp/花生壳等，另议）。

### 2.3 关联链（记住这条）

```
别人输入域名 linman.dynv6.net
   ↓ DNS 翻译成
你的公网 IP（123.45.67.89）
   ↓ 若是家庭宽带，还需路由器端口映射
内网服务器（192.168.1.100:8080）
```

域名本身只是"名字"，真正的落脚点是**公网 IP**（必要时再经端口映射到内网机器）。

## 三、步骤 3：在 dynv6.net 注册并申请免费域名

dynv6 提供免费的 `.dynv6.net` 二级域名，支持 IPv4（A 记录），申请简单，国内访问也还行。

### 3.1 注册账号
1. 浏览器打开 `https://dynv6.com`。
2. 点击右上角 **Sign in / Sign up**（注册登录在一起）。
3. 选 **Sign up**（注册），用邮箱注册，设置密码。
4. 去邮箱收验证邮件，点链接激活账号。
5. 回到 `https://dynv6.com` 登录。

### 3.2 创建你的免费域名（host）
1. 登录后进入控制台（Dashboard）。首次会看到提示"Don't have any hosts yet? Create one now."，点 **Create one now**。
2. 填写：
   - **Name（子域名前缀）**：填你想要的英文名，比如 `linman`。它将变成 `linman.dynv6.net`。
   - **Type**：选 **IPv4**（只要 A 记录；若也想支持 IPv6 可选 Both）。
   - **Location**：可留空。
3. 点击 **Create hostname**（创建）。
4. 创建成功，列表里出现 `linman.dynv6.net`。

### 3.3 拿到更新令牌（Token）
1. 在域名列表点你的 `linman.dynv6.net`，进入详情页。
2. 页面上有个 **Token**（一长串字符），这是以后 DDNS 客户端用来更新 IP 的"钥匙"。
3. **复制并保存好这个 Token**（后面步骤 4 要用）。

> 此时域名已存在，但还没指向正确的 IP（或指向 0.0.0.0）。下一步把它和你的公网 IP 绑定。

---

## 四、步骤 4：配置 DDNS，把公网 IP 与域名绑定更新

根据你 IP 是否固定，分两种做法。

### 4.1 情况一：你的公网 IP 是固定的（云服务器最常见）
**最简单：在网页上静态填一次就行，不用装任何客户端。**
1. 在 dynv6 域名详情页，找到 **IPv4 address** 一栏。
2. 填入你服务器的公网 IP（步骤 1 查到的那个，或云厂商控制台给的弹性公网 IP）。
3. 点 **Save（保存）**。
4. 等 1–2 分钟，浏览器打开 `http://linman.dynv6.net:8080`（前提是服务已起、端口已开），能访问即绑定成功。
5. **因为 IP 不变，以后不用再动它。**

### 4.2 情况二：你的公网 IP 会动态变化（家庭宽带）
需要让域名自动跟着 IP 变。原理：在服务器/常开机器上放一个小脚本，每隔几分钟把"当前公网 IP"汇报给 dynv6，dynv6 自动更新解析。

#### 方案（通用，Linux / 树莓派 / 服务器上跑）

1. 新建更新脚本 `update_dynv6.sh`：
   ```bash
   #!/bin/bash
   TOKEN="在这里粘贴你的dynv6 Token"
   HOST="linman.dynv6.net"
   # 不传 ipv4 参数时，dynv6 会自动检测你的出口 IP
   curl -s "https://dynv6.com/api/update?hostname=$HOST&token=$TOKEN"
   echo "  ($(date)) updated"
   ```
2. 给脚本加执行权限：
   ```bash
   chmod +x update_dynv6.sh
   ```
3. 先手动跑一次测试：
   ```bash
   ./update_dynv6.sh
   ```
   返回 `addresses updated` 或 `no changes` 都算成功。
4. 设成定时任务（每 5 分钟自动汇报）：
   ```bash
   crontab -e
   ```
   在最后加一行：
   ```bash
   */5 * * * * /绝对路径/update_dynv6.sh >> /绝对路径/dynv6.log 2>&1
   ```
   保存退出。以后 IP 一变，最多 5 分钟后域名自动跟上。

#### 方案（Windows 本机，适合用家里 Windows 当服务器）

1. 新建 `update_dynv6.bat`，内容：
   ```bat
   @echo off
   curl -s "https://dynv6.com/api/update?hostname=linman.dynv6.net&token=你的Token"
   ```
   （Win10/11 自带 `curl.exe`，可直接用。）
2. 打开「任务计划程序」→ 创建基本任务 → 触发器选"每天/重复"，设置每 5 分钟一次 → 操作选"启动程序"指向这个 .bat。
3. 保存后它会每 5 分钟自动运行，保持域名最新。

#### 验证 DDNS 是否生效
- 改个网络（比如手机切流量）后，浏览器开 `http://linman.dynv6.net:8080`，能通说明解析对了。
- 或在 cmd 跑 `nslookup linman.dynv6.net`，看返回的 IP 是否等于你当前公网 IP。

## 五、步骤 5：进阶操作（端口映射、服务部署、域名访问）

下面以「林蛮记账」项目为例，把服务真正跑起来并用域名访问。

### 5.1 路由器端口映射（仅家庭宽带场景 B 需要）
如果你是直接用云服务器（场景 A），**跳过这节**。
1. 浏览器开路由器后台（如 `192.168.1.1`），登录。
2. 找「端口转发 / 虚拟服务器 / Port Forwarding」。
3. 新增一条：
   - 外部端口：`8080`
   - 内部 IP：你那台内网服务器/电脑的 IP（如 `192.168.1.100`，建议在路由器给它绑固定内网 IP）
   - 内部端口：`8080`
   - 协议：TCP
4. 保存。现在外面访问 `公网IP:8080` 就会进到内网机器的 8080 服务。

### 5.2 在服务器上部署后端 + 前端（docker compose）
> 详细步骤见仓库 `DEPLOY_GUIDE.md` 和 `docs/deploy-experience.md`，这里只点关键：

1. 服务器装 Docker 和 Docker Compose。
2. 把 `docker-compose.yml` 传到服务器（首次需手动 `scp`）。
3. 服务器登录镜像仓库拉镜像（国内服务器建议用阿里云 ACR，ghcr 国内拉取慢，见坑 3.7）。
4. 准备服务器 `.env`，关键是这两行：
   ```env
   CORS_ORIGINS=http://localhost:5173,https://tauri.localhost,http://linman.dynv6.net:8080
   DB_HOST=db
   ```
   > 生产不要把 CORS 写成 `*`，把 App 源 `https://tauri.localhost` 和你的 Web 域名都列上。
5. 启动：`docker compose --env-file .env up -d`。
6. 确认存活：`curl http://localhost:8080/api/health` 返回 ok。

### 5.3 安卓 App 改用域名（必须重打包）
`VITE_API_BASE` 是打包时写死进 APK 的，改了必须重打：
1. 改 `frontend/.env`：
   ```env
   VITE_API_BASE=http://linman.dynv6.net:8080
   ```
2. 重跑 `build_android.bat`（等 `android-build.log` 末尾 `BUILD_EXIT_CODE=0`）。
3. 把新 APK 装到手机。App 里登录请求就会发往你的域名。

### 5.4 用域名访问
- **Web 端**：浏览器开 `http://linman.dynv6.net:8080`，应能看到前端页面。
- **App 端**：打开 App 登录，数据走 `http://linman.dynv6.net:8080` 到你的服务器。
- **验证 CORS**（可选）：电脑跑
  ```
  curl -i -X OPTIONS "http://linman.dynv6.net:8080/api/auth/login" -H "Origin: https://tauri.localhost"
  ```
  看响应头有无 `Access-Control-Allow-Origin: https://tauri.localhost`。

### 5.5 关于备案与端口（重要）
- 国内服务器用 **80/443 端口 + 域名** 提供 Web 服务，**必须 ICP 备案**（备案本身免费，但需实名、约 1–2 周）。
- 本文用 **8080 高位端口 + 免费子域名**，个人自用一般**不强制备案**，零成本即可跑。
- 想做到 `https://linman.dynv6.net`（无端口、小绿锁）那种"好看"形态，需要：买个自己的域名 + 国内备案 + 配 TLS。那是下一步付费升级，本文先不展开。

## 六、常见问题排错

| 现象 | 可能原因 | 解决 |
|------|----------|------|
| 浏览器开 `域名:8080` 打不开 | 服务没起 / 端口没开 / 防火墙拦了 | 服务器上 `curl http://localhost:8080/api/health` 先确认服务活；云服务器安全组放行 8080；家庭宽带确认端口映射和路由器防火墙 |
| 域名解析到的 IP 不是我的 | DDNS 没更新 / Token 错 | 重跑更新脚本看返回；核对 Token；`nslookup 域名` 对比公网 IP |
| 家里宽带没有公网 IP（WAN 是 100.64.x / 192.168.x） | 运营商 NAT | 打电话给运营商要公网 IP；要不改用内网穿透（frp/花生壳） |
| App 登录被 CORS 拦 | 服务器 CORS_ORIGINS 没加源 | 在服务器 `.env` 的 `CORS_ORIGINS` 加上 `https://tauri.localhost` 和你的域名，重启后端容器 |
| App 还是连旧地址 | APK 没重打包 | `VITE_API_BASE` 改后必须重跑 `build_android.bat` 重装 APK |
| 域名能开但过会儿连不上 | 家庭宽带 IP 变了但 DDNS 没跟上 | 确认更新脚本/cron/任务计划真的在跑（看日志） |

## 七、一句话总结流程

```
查公网 IP（步骤1）
   → 判断 IP 是否固定、是否有公网 IP（步骤2）
   → dynv6 注册 + 申请域名 + 拿 Token（步骤3）
   → 固定 IP 就网页静态填；动态 IP 就装更新脚本（步骤4）
   → 端口映射（家庭宽带）+ docker 部署 + App 改域名重打包 + 用域名访问（步骤5）
```

照这 5 步走，零成本就能用 `http://linman.dynv6.net:8080` 替代难看的 IP 地址，让国内任何有网的人都能访问你的服务。
