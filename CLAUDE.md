# 林蛮记账 — 产品文档

## 📌 产品概述

- **产品名称**：林蛮记账
- **产品定位**：个人日常记账桌面应用，记录收入与支出
- **运行平台**：Windows 10/11 + macOS（Intel & Apple Silicon）
- **货币单位**：人民币（元）
- **目标用户**：普通个人用户，用于掌握自己的收支情况

---

## 🎯 核心功能

### V1.0 版本

1. **记录收支**：用户可记录每一笔账目，包含：
   - 类型（支出 或 收入）
   - 金额（人民币，精确到分，如 12.50 元）
   - 日期（默认为当天，可修改）
   - 分类选择（二级分类，根据类型动态切换）
   - 备注（可选，文字描述）

2. **二级分类体系**：每笔收支归属一个一级大类 + 二级小类（支出分类 和 收入分类 各自独立）

3. **收支列表**：查看所有记录，支持按时间排序、按类型筛选（只看支出/只看收入/全部）

4. **收支概览**：首页显示本月收入总额、支出总额、结余

4. **数据持久化**：数据保存在本地电脑，关闭APP再打开数据不丢失

## 🛠 技术栈

| 层面 | 技术选型 | 说明 |
|------|---------|------|
| 桌面框架 | **Tauri v2** | ✅ 已迁移 — 比 Electron 更轻量（安装包 3-5MB），原生性能 |
| 后端语言 | Rust | Tauri 后端，处理系统调用和数据库 |
| 前端框架 | React | 构建用户界面的 JavaScript 框架 |
| UI 组件库 | **Ant Design 5.x** | ✅ 已确认 — 国内最流行，中文文档一流 |
| 数据存储 | **SQLite (sqlx)** | ✅ 已确认 — 通过 Tauri SQL Plugin 实现 |
| 界面布局 | **侧边栏布局** | ✅ 已确认 — 左侧导航，右侧内容 |
| 构建工具 | Vite + Tauri CLI | 快速的开发构建工具 |
| 打包工具 | Tauri Bundler | 打包成 Windows .exe/.msi 和 Mac .dmg |
| 开发语言 | TypeScript + Rust | 前端 TypeScript，后端 Rust |

> **为什么从 Electron 换成 Tauri？**
> Windows 上 Electron 有一个已知 Bug (#49034)，导致核心模块无法加载。Tauri 没有这个问题，而且更轻更快。

## ⚠️ 重要项目规则（全项目期遵守）

> 本项目的用户为非技术背景小白，对编程技术不了解。
>
> **因此，在整个项目开发过程中，任何涉及技术决策的事项，开发者必须：**
>
> 1. **列出至少 2-3 个可行方案**
> 2. **用通俗易懂的大白话解释每个方案的优劣**
> 3. **给出推荐方案并说明理由**
> 4. **让用户做出最终选择后，再继续执行**
>
> ❌ 禁止行为：不做任何技术假设，不在用户未选择前直接敲定技术方案
> ✅ 正确做法：遇到选择 → 列方案 → 解释优劣 → 等用户决定 → 执行
